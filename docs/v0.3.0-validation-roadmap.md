# v0.3.0 Validation Roadmap

Date: 2026-05-31

## Goal

Move from feature-complete MVP behavior to a validated engineering analysis core. The v0.3.0 target is confidence in signal-analysis correctness, not GUI, DAQ, RTOS, Zephyr, production signal integrity, or certification support.

## Milestone

GitHub milestone: `M4: Signal Accuracy and Validation`

Recommended release label: `v0.3.0 = validated engineering analysis core`

## Validation Dataset Layout

```text
validation/
|-- known_answer/
|-- environmental_cases/
|-- filter_response/
`-- reports/
```

Each dataset must include:

- Source fixture or generation notes.
- Expected measurement values.
- Tolerance policy.
- Analyzer command.
- Expected report artifact.
- Review notes that state what the dataset proves and what it does not prove.

## Sequencing

1. Finish M1 metadata and README expected-output issues.
2. Keep M3 RTOS and Zephyr issues open but deprioritized.
3. Add known-answer validation before adding GUI, bindings, DAQ integrations, or RTOS adapters.
4. Add tolerances before relying on floating-point criteria for engineering review.
5. Document filter equations before claiming behavior beyond MVP smoothing.
